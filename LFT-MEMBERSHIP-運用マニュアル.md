# LFT会員プラグイン（lft-membership）運用メモ

キャンペーンや運用の変更に合わせて、事務局側で触りそうなところだけ抜き出してあります。分からないところは無理に触らず、開発担当に声をかけてください。

---

## 変更を反映するまでの流れ（ざっくり）

まず手元でファイルを直し、サーバーに載せ替えます。PHP なので、サーバーに上がった時点で次の送信・次の表示から反映されます。ビルドのような作業はありません。

編集するのは、だいたい次のフォルダの中です。

`wp-content/plugins/lft-membership/`

載せ替え方は、FTP で上書きしてもいいですし、後述のとおり ZIP でまとめて入れ替えても構いません。どちらにせよ、**本番に出す前にバックアップ**を取っておくのが安全です。

---

## ZIP ファイルでプラグインを入れ替える方法

管理画面からプラグインを丸ごと差し替えたいときの手順です。ローカルでメール文面などを直したあと、フォルダごと ZIP にしてアップロードする想定です。

### ZIP の作り方（パソコン側）

1. `lft-membership` という名前のフォルダを用意します（中身はプラグイン一式）。
2. そのフォルダをそのまま圧縮して ZIP にします。  
   Windows ならフォルダを右クリック → 送る → 圧縮（zip 形式）など。

ここでよくある失敗は、ZIP を開いたときに **いきなり php ファイルが並んでいる** 状態になっていることです。正しくは、ZIP の一番上に **`lft-membership` フォルダが一つ**あり、その中に `lft-membership.php` などが入っている形です。WordPress はこの形を前提にしています。

### 管理画面での入れ替え

1. 管理画面にログインし、プラグインの一覧を開きます。
2. いま入っている「lft-membership」を**無効化**します。
3. 無効化したあと、**削除**します（中身のファイルがサーバーから消えます。本番では事前バックアップ推奨です）。
4. 「プラグイン」→「新規追加」→「プラグインのアップロード」から、先ほど作った ZIP を選び、インストールします。
5. **有効化**します。

会員の登録情報はデータベース側にあり、プラグインのフォルダを消しただけでは消えない想定ですが、運用ルールや将来の仕様変更で挙動が変わることもあるため、**本番で削除する前にバックアップを取る**か、不安なときは開発担当に一度確認してください。

## メール文面を変えるとき

どのメールを直したいかを決めたら、下の表でファイルと、だいたいの行をたどります。件名と本文は別のブロックに書いてあるので、両方変える場合は両方探してください。

行番号はバージョンで前後します。迷ったらファイル名と関数名（表に書いてある英語）で検索すると早いです。

`{$name}` のような **波括弧付き** はシステムが後から埋める部分なので、消したり中身を変えたりしないでください。

| どのメールか | ファイル | 件名（目安） | 本文（目安） |
|--------------|----------|--------------|--------------|
| 新規会員向け・パスワード設定 URL の案内（管理者が会員を追加したとき） | `includes/class-lft-membership-registration-email.php` | 冒頭付近の `lft_membership_registration_invite_subject` のところ | `build_body` の中（長い日本語のブロック） |
| パスワードを忘れたときの再設定案内 | `includes/class-lft-membership-frontend.php` | `lft_membership_password_reset_email_subject` 付近 | `get_password_reset_email_body` の中 |
| 初回登録が終わった直後の長い案内メール | 同上 | `lft_membership_registration_confirmation_subject` 付近 | `get_registration_confirmation_email_body` の中 |
| パスワードを変え終わったときの完了メール | 同上 | `lft_membership_password_changed_email_subject` 付近 | `get_password_changed_email_body` の中 |
| メールアドレス変更の承認依頼（いま登録されているメール宛） | 同上 | `lft_membership_email_change_confirm_subject` 付近 | `get_email_change_confirmation_email_body` の中 |
| メールアドレス変更が終わった通知（新しいメール宛） | 同上 | `lft_membership_email_changed_notice_subject` 付近 | `get_email_changed_notice_email_body` の中 |
| 会員期限の前日お知らせ | 同上 | `lft_membership_expiration_reminder_subject` 付近 | `get_expiration_reminder_email_body` の中 |
| 事務局向け・メール変更が完了したときの通知 | 同上 | `notify_office_email_change_completed` まわり | `get_office_email_change_completed_body` の中 |

送信者名や送信元メールを変えたい場合は、同じ `class-lft-membership-frontend.php` の前半、`get_plugin_mail_headers` のあたりに、`lft_membership_mail_from_name` と `lft_membership_mail_from_email` という名前の設定があります。キャンペーンで差出人表示だけ変える、といったときはここを見ることが多いです。

## 通知まわり（BCC・事務局宛）

管理画面に「BCC を切る」「通知先を変える」といった専用の画面はありません。プログラム側の「フィルター」で決まっています。

会員宛のメールと同じ文面を事務局にも送っている処理があり、デフォルトでは事務局側のアドレスがコード上に書かれています。細かく変えたい場合は、プラグインを直接いじるより、子テーマの `functions.php` に短い追記をする方が安全なことが多いです。ただ、ここを間違えると通知が来なくなるので、**本番で触る前に開発担当に相談**するのがおすすめです。

## ログインが効く範囲（どのページで LFT ログインに差し替えるか）

ここは **管理画面のクリックだけでは広げたり狭めたりできません**。URL の構造やプログラムの修正が絡むので、対象を増やしたい・外したいときは **開発担当への依頼が必要**です。

もともとこのプラグインは、どのサイトにも入れられるように作ってありました。今回のサイトでは、ご要望に合わせて **「パスワード保護」の挙動を差し替えるのは `/lft_membership/` 配下に限る** 形に寄せています。コードでは「そのページの URL が会員用のスラッグから始まるか」を見ています。

ページを WordPress で増やすだけでは足りないこともあり、公開範囲やログインまわりの不具合につながりやすいので、仕様を変えたくなったら開発側に一度話を通してください。
